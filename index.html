SpanPro / ReadOnlySpanPro

Span kullanmadan yüksek performanslı, güvenli ve esnek bir dilimleme (slicing) kütüphanesi. Tek tip ile Array / String(RO) / Unmanaged bellek arka planlarını kapsar; arama, kopyalama, biçimleme ve metin üretiminde düşük GC ve yüksek hız hedefler.

Array / String(RO) / Unmanaged BMH hızlandırmalı arama Karakter-bazlı ToString StringRO yazılamaz MIT License
Namespace: DSO.Core.SpanPro Tipler: SpanPro<T>, ReadOnlySpanPro<T> Kind: Array · StringRO · Unmanaged

Neden SpanPro?

  • Span’sız performans: .NET Span<T> kullanmadan benzer ergonomi + hız yaklaşımı.
  • Çoklu backing, tek tip: owner + start + length + kind modeliyle Array/String(RO)/Unmanaged tek yapı altında.
  • Özel algoritmalar: char/byte alt dizi aramalarında BMH (Boyer–Moore–Horspool) ile güçlü hızlanma.
  • GC-dostu: ArrayPool, stackalloc ve iki-geçişli yazım; ara tahsisatlardan kaçınır.
  • Deterministik kimlik: Equals/GetHashCode → owner + start + length + kind.

Öne Çıkan Özellikler

SpanPro<T> (yazılabilir) ve ReadOnlySpanPro<T> (salt okunur).

IndexOf, LastIndexOf, FindFirst/Last, FindAll (+ BMH).

Tek buffer + TryFormat ile GC-free yazım.

Mimari

SpanPro, “span benzeri” bir görünümü owner + start + length + kind modeliyle taşır. Erişim sırasında kalıcı pointer tutmak yerine ItemRef / ItemRefRO ile gerekli ref/pointer anlık hesaplanır.

  • Array: yazılabilir / salt okunur yollar.
  • StringRO: salt okunur; yazma denemesi exception.
  • Unmanaged: IRawMemoryOwner üzerinden; referans içeren T yasak.
Raw memory sahipliği
Repo içinde unmanaged/pinli bellek için basit sahiplik modeli bulunur: IRawMemoryOwner, PinnedArrayOwner ve UnmanagedOwner.

Semantik & Güvenlik

  • String arkasına yazma yasak: StringRO backing’de ItemRef çağrısı InvalidOperationException üretir.
  • Unmanaged + referans içeren T yasak: GC taramadığı için NotSupportedException.
  • Out-of-range: ArgumentOutOfRangeException (slice/kopyalama/erişim).
  • Eşitlik: owner + start + length + kind aynıysa eşit.
Not: Unmanaged kullanımında IRawMemoryOwner yaşam döngüsü çok kritiktir. Owner dispose edildikten sonra span görünümünü kullanmayın.

ToString — Karakter Bazlı Semantik

ToString(int start, int length) tüm türlerde “eleman sayısı” değil “toplam karakter akışı” üzerinden dilimler.

  • T == char: start/length zaten karakterdir (array’de hızlı yol: new string(char[], start, length)).
  • T == string: string[] parçaları tek bir metinmiş gibi düşünülür, sonra karakter aralığı uygulanır.
  • Diğer T: her eleman yazıya çevrilir (önce ISpanFormattable.TryFormat, olmazsa ToString()), sonra karakter aralığı uygulanır.
  • Kültür: varsayılan CultureInfo.CurrentCulture.
// Örnekler
// long[] { 1, 23, 456 }  -> ToString(0,6)  == "123456"
// string[] { "abc", "def", "g" } -> ToString(2,4) == "cdef"

Hızlı Başlangıç

using DSO.Core.SpanPro;

// 1) Dizi ile başla
var arr = new int[] { 10, 20, 30, 40, 50 };
var sp  = SpanPro<int>.FromArray(arr);         // yazılabilir
var ro  = ReadOnlySpanPro<int>.FromArray(arr); // salt okunur

// Dilimleme & yazma
var mid = sp.Slice(1, 3); // [20,30,40]
mid.Fill(99);             // arr => {10, 99, 99, 99, 50}

// Arama
int ix = ro.IndexOf(99, 0);
int lx = ro.LastIndexOf(99);
var all = ro.FindAll(99, 0); // {1,2,3}
// String (RO) üzerinde alt dizi arama + slicing + ToString
var s   = "lorem ipsum lorem";
var rs  = ReadOnlySpanPro<char>.FromString(s);
var pat = ReadOnlySpanPro<char>.FromString("ipsum");

int where = rs.IndexOf(pat, 0);            // BMH destekli
string cut = rs.Slice(where, 5).ToString(0, 5); // "ipsum"
// Unmanaged örneği
using var um = new UnmanagedOwner<byte>(1024);
var ub = SpanPro<byte>.FromUnmanagedOwner(um, 0, 1024);

ub.Fill(0x20);
ub.ItemRef(100) = 0x2A;

int pos = ub.IndexOf((byte)0x2A, 0);

var dst = new byte[1024];
ub.CopyTo(dst, 0);
Kurulum (önerilen)
Bu repo şu an “source-first” kullanıma uygundur: projene Project Reference olarak ekleyebilir veya kaynakları doğrudan dahil edebilirsin. (Repo sayfasında release/NuGet bilgisi görünmüyor.)

Arama Ailesi

  • Tek değer: IndexOf(value, start), LastIndexOf(value)
  • Alt dizi: IndexOf(pattern, start), LastIndexOf(pattern, end)
  • İlk/Son: FindFirst, FindLast
  • Çoklu: FindAll(value, start) → int[] (ArrayPool ile büyütmeli)
  • BMH: char/byte için uzun patternlerde agresif hızlanma
Performans ipucu
Sıcak döngülerde kind dallanması minimize edilmiştir; Array/Unmanaged yollarında ref/pointer ile ilerleme hedeflenir.

Kopyalama & Grow

  • CopyTo: aynı owner’da Array.Copy / Buffer.MemoryCopy hızlı yolları.
  • DsMemoryCopySP: overlap-safe, write-barrier uyumlu genel kopya yolu.
  • Grow: ArrayPool ile kapasite artırma yaklaşımı (mantıksal uzunluk vs kapasite notlarını takip edin).
Grow notu: Eğer “Length == kapasite” davranışı kullanılıyorsa, API’yi tüketen kodda “mantıksal uzunluk” bilgisini ayrıca yönetmek gerekebilir. (Özellikle buffer builder senaryolarında.)

NumberFormatting (GC-free yazım)

Sayıları ara string oluşturmadan hedef SpanPro<char> üzerine yazmak için NumberFormatting yardımcıları bulunur. Alt tarafta minimal örnek: 

var buf = new char[64];
var txt = SpanPro<char>.FromArray(buf);

int w = 0;
w += NumberFormatting.FormatInt64(ref txt, w, -12345,
     NumberFormatSlim.FromSeparators(',', '-'));

w += NumberFormatting.FormatDouble(ref txt, w, 3.14159,
     NumberFormatSlim.Invariant, "F3");

string sOut = txt.ToString(0, w); // "-12,3453.142"
Not: Format işlemleri içeride TryFormat + stackalloc geçici buffer kullanır, ardından hedefe kopyalar; amaç GC baskısını düşürmektir.

Mini API Kılavuzu

// Oluşturma
SpanPro<T>.FromArray(T[] arr, int start = 0, int length = arr.Length)
ReadOnlySpanPro<T>.FromArray(T[] arr, int start = 0, int length = arr.Length)
ReadOnlySpanPro<char>.FromString(string s, int start = 0, int length = s.Length)
SpanPro<T>.FromUnmanagedOwner(IRawMemoryOwner<T> owner, int start, int length)

// Erişim
ref T            ItemRef(int index);      // yazılabilir (StringRO'da exception)
ref readonly T   ItemRefRO(int index);    // salt okunur

// Kullanışlılar
Slice(int start, int length)
ToArray(int start, int length, bool isRemoveNull = false)
CopyTo(T[] destination, int destStart)
Fill(value) / Clear()
IndexOf / LastIndexOf / FindAll / IndexOf(pattern)
ToString(int start, int length) // tüm türlerde karakter bazlı
Extension’lar
Repo içinde SpanPro.Extensions ile pratik yardımcılar bulunur: örn. string’i yazılabilir char[] kopyasına çeviren AsSpanProMutable ve trim yardımcıları.

Kullanım Senaryoları

  • Parser / tokenizer: slicing + arama + düşük tahsisat.
  • Metin üretimi: karakter-bazlı ToString ve buffer üstüne yazım.
  • Binary protokol işleme: unmanaged buffer + hızlı arama/kopyalama.
  • String üzerinde güvenli okuma: ReadOnlySpanPro<char> ile string backing.
Memory/Span ile fark
BCL’in Memory/Span ekosistemine doğrudan bağlanmak yerine, Span’sız aynı problemi çözmeye odaklanır; özellikle string backing ve kütüphane içi BMH/overlap-safe copy gibi özelleştirmeler öne çıkar.
